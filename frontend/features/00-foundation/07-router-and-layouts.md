# 00-07 Router, auth guard и layouts

## Цель

Настроить все маршруты приложения, auth guard и два layout'а (AppLayout с navbar, AuthLayout для страниц входа).

## Зачем

Маршруты определяют навигационную структуру всего приложения. Auth guard защищает страницы кабинета от неавторизованных и редиректит их на логин. Два layout'а нужны: один с полноценным navbar для основных страниц, другой — минималистичный для форм входа/регистрации. Страницы пока создаются как заглушки — они будут наполнены в следующих фазах.

## Use Cases

- Неавторизованный пользователь открывает `/sandbox` — работает полноценно, без логина
- Неавторизованный пользователь открывает `/templates` — видит публичную библиотеку
- Неавторизованный пользователь переходит на `/dashboard` — оказывается на `/login?redirect=/dashboard`
- После логина — возвращается на `/dashboard` через query-параметр
- Авторизованный пользователь идёт на `/login` — редиректится на `/sandbox`
- Страницы подгружаются лениво (lazy load) — начальный бандл не раздувается

## Критерии готовности

- [ ] Все маршруты зарегистрированы: `/`, `/login`, `/register`, `/forgot-password`, `/change-password`, `/sandbox`, `/s/:id`, `/templates`, `/dashboard`, `/dashboard/templates`, `/dashboard/runs`
- [ ] Guard работает: защищённые маршруты → `/login`, guestOnly маршруты → `/sandbox`
- [ ] AppLayout отображает navbar с ссылками и user-меню
- [ ] AuthLayout — центрированная карточка без navbar
- [ ] Все страницы-заглушки созданы (пустые компоненты, без ошибок)
- [ ] `npm run build` без ошибок
