# 00-04 Mock-инфраструктура

## Цель

Создать mock-реализации API-модулей для работы без бэкенда в dev-режиме. Движки — хардкод на фронте, моки не нужны.

## Зачем

Фронтенд разрабатывается параллельно с бэкендом. Моки позволяют работать с реальными данными в UI без поднятого сервера. При `__USE_MOCK__ === true` каждый API-вызов возвращает заготовленный ответ с задержкой 100–200ms — имитация сети.

## Use Cases

- Разработчик открывает страницу шаблонов — видит список тестовых шаблонов без бэкенда
- Разработчик логинится — mock возвращает фиктивного пользователя, store обновляется
- Разработчик удаляет шаблон — mock мутирует внутренний массив, список обновляется корректно
- Разработчик сохраняет состояние sandbox — mock возвращает фиктивный id

## Критерии готовности

- [ ] Моки созданы для: auth, templates, render-runs, state
- [ ] Все моки возвращают типизированные данные, совпадающие с `src/types/index.ts`
- [ ] Задержки присутствуют (имитация реальной сети)
- [ ] Мутирующие операции (create/delete/clone) отражаются в последующих get-запросах внутри сессии
- [ ] Smoke test: вызвать `getMyTemplates()` в dev-режиме из консоли браузера — возвращает массив без сетевого запроса в Network tab
