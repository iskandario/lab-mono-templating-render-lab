# 00-03 HTTP клиент

## Цель

Единый fetch-wrapper для всех API-запросов с поддержкой session cookie и автоматическим logout при 401.

## Зачем

Все запросы к бэкенду должны идти через одно место. Авторизация работает через session cookie (?) (не JWT) — браузер отправляет его автоматически при `credentials: include`. При протухшей сессии (401) пользователь автоматически уходит на /login, не получая сломанного UI.я

Бэкенд ещё в разработке — URL endpoint'ов могут меняться. Все пути хранятся в одном конфиге `src/api/endpoints.ts`, API-модули импортируют оттуда. Менять URL = менять один файл, не искать по всем модулям.

## Use Cases

- Разработчик вызывает `http.get(ENDPOINTS.templates.list)` — не думает про base URL, credentials и обработку ошибок
- Бэкенд переименовал `/auth/login` в `/auth/sign-in` — правка в одном месте, всё работает
- Сессия протухла во время работы — пользователь уходит на /login автоматически
- Бэкенд вернул 4xx/5xx — выбрасывается Error с текстом из ответа

## Критерии готовности

- [ ] Файл `src/api/endpoints.ts` содержит все URL paths, сгруппированные по домену (auth, templates, renderRuns, state)
- [ ] API-модули используют `ENDPOINTS.*`, не хардкодят строки путей
- [ ] Все запросы приходят с `credentials: include`
- [ ] При 401 — вызывается logout store и редирект на /login
- [ ] Ошибки 4xx/5xx превращаются в Error с human-readable сообщением
- [ ] `npm run type-check` без ошибок
