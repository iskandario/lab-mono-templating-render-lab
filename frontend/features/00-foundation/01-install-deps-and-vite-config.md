# 00-01 Установка зависимостей и Vite конфиг

## Цель

Установить pinia, настроить dual Vite config (prod / dev+mock), объявить глобальный флаг `__USE_MOCK__` для моков, чтобы не ждать бэк.

## Зачем

Dev-режим должен работать без запущенного бэкенда. Флаг `__USE_MOCK__` позволяет API-модулям переключаться между моками и реальными запросами в одном месте — без env-переменных и без изменения кода перед деплоем. Сами моки - позже.

## Use Cases

- Разработчик запускает `npm run dev` — приложение работает с моками, нет ошибок сети
- Разработчик запускает `npm run build` — собирается продакшн без моков
- TypeScript не ругается на использование `__USE_MOCK__` в коде

## Критерии готовности

- [ ] `npm install` проходит без ошибок
- [ ] `npm run dev` стартует и открывает приложение
- [ ] `npm run build` собирается без ошибок
- [ ] В коде можно написать `if (__USE_MOCK__) ...` без TS-ошибок

## Зависимости

Нет
