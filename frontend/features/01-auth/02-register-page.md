# 01-02 Страница регистрации

## Цель

Форма регистрации по email + password с подтверждением пароля. После успешной регистрации — автоматический вход.

## Use Cases

- Пользователь заполняет форму, нажимает "Зарегистрироваться" — попадает на sandbox, уже авторизованным
- Пароли не совпадают — форма не отправляется, видна ошибка валидации
- Email уже занят — бэкенд возвращает ошибку, пользователь видит её под формой

## Критерии готовности

- [ ] Три поля: email, password, confirm password
- [ ] Валидация: email-формат, min 8 символов, совпадение паролей
- [ ] Ошибка API отображается пользователю
- [ ] После успешной регистрации — редирект на `/sandbox`
- [ ] Ссылка на `/login`
