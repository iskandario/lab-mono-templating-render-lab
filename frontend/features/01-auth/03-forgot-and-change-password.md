# 01-03 Сброс и смена пароля

## Цель

Две независимые формы: сброс пароля по email (для гостей) и смена пароля из настроек (для авторизованных). Пока не делаем

## Use Cases

**Сброс пароля:**
- Пользователь ввёл email и отправил форму — видит сообщение "Проверьте почту" (не важно, существует ли email — безопасность)
- Пользователь вводит невалидный email — кнопка недоступна

**Смена пароля:**
- Авторизованный пользователь меняет пароль — видит успешное уведомление, остаётся на странице
- Вводит неверный текущий пароль — видит ошибку от бэкенда
- Новый пароль совпадает со старым — клиентская ошибка, не отправляется

## Критерии готовности

- [ ] ForgotPasswordPage: после отправки показывает успех-сообщение
- [ ] ChangePasswordPage: маршрут `/change-password` требует авторизации (guard)
- [ ] Валидация: новый пароль ≠ текущий, совпадение с подтверждением, min длина
- [ ] Ошибки API отображаются
