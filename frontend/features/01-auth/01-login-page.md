# 01-01 Страница логина

## Цель

Форма входа по email + password с клиентской валидацией и отображением ошибок от бэкенда.

## Зачем

Точка входа в приложение. После успешного логина пользователь попадает туда, куда хотел (через `?redirect=`) или на `/sandbox`.

## Use Cases

- Пользователь вводит правильные данные — попадает на sandbox (или на страницу, с которой его ушёл guard)
- Пользователь вводит неверный пароль — видит сообщение об ошибке под формой
- Пользователь не заполнил поле — кнопка Submit недоступна
- Пользователь кликает "Забыли пароль?" — переходит на `/forgot-password`

## Критерии готовности

- [ ] Валидация email-формата и минимальной длины пароля на клиенте
- [ ] Submit недоступен при невалидной форме
- [ ] Loading-состояние кнопки во время запроса
- [ ] Ошибка от API отображается пользователю
- [ ] После успешного логина — редирект на `?redirect` или `/sandbox`
- [ ] Ссылки на `/register` и `/forgot-password`
