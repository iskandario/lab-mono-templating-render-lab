# 03-02 BenchmarkRunner UI и MetricsPanel

## Цель

UI для запуска бенчмарка (выбор N, кнопка Run, прогресс, Cancel) и MetricsPanel с реальными данными в editor и compare режимах. Сохранение результата на бэкенд — только по явному действию пользователя.

## Use Cases

- Пользователь выбирает или вводит 1000 итераций, нажимает Run — видит progress bar, кнопки заблокированы
- Бенчмарк завершается — MetricsPanel показывает avg/min/max/p95 и размер вывода
- Пользователь нажимает Cancel — бенчмарк прерывается
- В compare mode MetricsPanel показывает две колонки: Template A и Template B side-by-side с процентом разницы
- "Template B faster by 23%" вычисляется автоматически по avg
- Пользователь нажимает "Сохранить запуск" — результат отправляется на бэкенд через `saveRun()`, появляется в истории My Runs. Без нажатия — никаких запросов к бэкенду.

## Критерии готовности

- [ ] Поле ввода и подсказка: 100, 500, 1000, 5000 итераций
- [ ] Run запускает бенчмарк для активного слота (editor mode) или обоих (compare mode)
- [ ] Progress bar обновляется в реальном времени
- [ ] Cancel доступен во время выполнения
- [ ] MetricsPanel в editor mode — метрики активного слота; в compare mode — две колонки
- [ ] Compare: "Template X faster by Y%" или "Identical speed"
- [ ] Форматирование: `1.23ms`, `45μs`, `1.2KB`
- [ ] Кнопка "Сохранить запуск" в action bar (для авторизованных) — единственная точка отправки на бэкенд
- [ ] Без нажатия "Сохранить запуск" никаких запросов на бэкенд не происходит
