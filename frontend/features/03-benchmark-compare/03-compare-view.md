# 03-03 CompareView — split preview

## Цель

Split-preview в compare mode: два iframe рядом с названиями движков и сравнением вывода.

## Зачем

Визуальное сравнение двух шаблонов — центральная фича compare mode. Пользователь видит не только метрики, но и сам вывод рядом. Построчный diff показывает, идентичен ли вывод двух движков.

## Use Cases

- Пользователь переключается в Compare mode — Preview разделяется на две части с заголовками "Template A · Handlebars" и "Template B · Pug"
- Оба шаблона рендерят одинаковый HTML — зелёный чип "Output: identical"
- Выводы отличаются — жёлтый чип "N lines differ"
- Бенчмарк ещё не запускался — в панелях плейсхолдер "Run benchmark first"
- Пользователь возвращается в Editor mode — всё состояние сохранено

## Критерии готовности

- [ ] Два iframe рядом (50/50), каждый показывает output соответствующего слота
- [ ] Заголовки с именами движков
- [ ] "Output: identical" при полном совпадении строк
- [ ] "N lines differ" при различиях (построчное сравнение)
- [ ] Плейсхолдер при отсутствии результатов бенчмарка
- [ ] Переключение Editor ↔ Compare не теряет состояние
