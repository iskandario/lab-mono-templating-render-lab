# 03-01 Benchmark service и metrics calculator

## Цель

Сервис запуска N итераций рендера с замером времени через `performance.now()` и утилита расчёта статистик (avg/min/max/p95).

## Зачем

Бенчмарк — ключевая фича платформы. Всё выполняется на клиенте: нет нагрузки на сервер, результаты отражают реальную производительность движка в браузере пользователя. Yield через `setTimeout(0)` каждые 50 итераций — UI не замерзает даже при 5000 итерациях. Первые 5 итераций — warmup и не учитываются (cold start движка искажает результат).

## Use Cases

- Пользователь запускает 1000 итераций — UI остаётся отзывчивым, progress bar обновляется
- Пользователь нажимает Cancel — бенчмарк останавливается немедленно
- Шаблон содержит ошибку — бенчмарк останавливается с описанием ошибки
- Метрики рассчитываются корректно: p95 — 95-й перцентиль отсортированного массива таймингов

## Критерии готовности

- [ ] N итераций выполняются с yield каждые 50 (нет заморозки UI)
- [ ] 5 warmup-итераций не входят в статистику
- [ ] Отмена через AbortSignal работает
- [ ] Рассчитываются: avg, min, max, p95, outputBytes
- [ ] Ошибка рендера останавливает бенчмарк с ошибкой
- [ ] `npm run type-check` без ошибок
