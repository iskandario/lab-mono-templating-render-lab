# 02-02 Monaco Editor wrapper

## Цель

Vue-компонент-обёртка над Monaco Editor: lazy load, v-model для кода, смена языка подсветки при смене движка.

## Зачем

Monaco — тяжёлый (~2MB). Грузить его лениво и один раз. Обёртка скрывает сложность инициализации: снаружи компонент выглядит как обычный `v-model`-инпут с пропом `language`. При смене движка меняется только подсветка и контент, редактор не пересоздаётся.

## Use Cases

- Пользователь видит Monaco Editor с подсветкой синтаксиса для handlebars
- Пользователь меняет движок на pug — подсветка меняется без мерцания и потери курсора
- Пользователь изменяет размер окна — редактор адаптируется под контейнер
- Страница закрывается — редактор корректно уничтожается (нет утечек)

## Критерии готовности

- [ ] `npm install @monaco-editor/loader`
- [ ] Monaco загружается лениво (не блокирует начальный рендер)
- [ ] `v-model` работает: правки в редакторе → emit, изменение пропа → редактор обновляется
- [ ] Смена `language` пропа → меняется подсветка без пересоздания редактора
- [ ] Dispose при `onBeforeUnmount`
- [ ] Редактор занимает 100% контейнера и автоматически ресайзится
