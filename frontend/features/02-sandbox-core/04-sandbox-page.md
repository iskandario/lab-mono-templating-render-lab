# 02-04 SandboxPage — сборка

## Цель

Собрать все sandbox-компоненты в одну страницу: двухпанельный layout (редактор слева, preview+metrics справа) и action bar снизу.

## Layout

```
┌──────────────────────────────────────────────────────┐
│ [Tab A ▼engine] [Tab B ▼engine] [JSON]               │
├─────────────────────┬────────────────────────────────┤
│                     │  Preview (~2/3 высоты)          │
│   Monaco Editor     ├────────────────────────────────┤
│                     │  Metrics (~1/3 высоты)          │
├─────────────────────┴────────────────────────────────┤
│ [Run Benchmark] [Compare] [N]      [Share] [Save]    │
└──────────────────────────────────────────────────────┘
```

## Зачем

Sandbox — главная страница приложения. Все компоненты собираются здесь. Движки — хардкод из engines store (не запрос к API). Кнопки Save/Share в action bar отправляют данные на бэкенд только по явному нажатию пользователя — не автоматически.

## Use Cases

- Пользователь открывает `/sandbox` — сразу видит редактор с подсветкой, preview с рендером дефолтного шаблона (Hello World)
- EngineSelector заполнен хардкодным списком движков
- Пользователь нажимает Save — состояние уходит на бэкенд (если авторизован) или в clipboard (гость)

## Критерии готовности

- [ ] Двухпанельный layout без горизонтального скролла
- [ ] Sandbox занимает всю высоту viewport минус navbar
- [ ] Live preview работает сразу при открытии
- [ ] Action bar присутствует: Run Benchmark, Compare toggle, N selector, Share, Save
- [ ] `npm run build` без ошибок
