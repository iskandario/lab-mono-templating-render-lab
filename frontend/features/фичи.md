# Полный список фичей

## 1. Авторизация

- Регистрация и логин (`email` + `password`)
- Сброс пароля
- Смена пароля в настройках

## 2. Личный кабинет

- Мои шаблоны: список, создание, удаление, клонирование
- Мои RenderRun-запуски: история с метриками
- Настройки профиля: пароль

## 3. Sandbox / редактор

### Архитектура рендера

- Рендер — клиентский (JS-бандлы движков в браузере)
- Бенчмарк — N итераций на клиенте через `performance.now()` → avg / min / max / p95
- Бэкенд получает только результат для сохранения (RenderRun), сам не рендерит
- Список доступных движков подтягивается с бэкенда — фронт не хардкодит

### Layout (CodeWars-style)

[посмотреть на попытку прототипирования](https://www.figma.com/design/MiVVtPqZQVvjxdHpxYJtit/Untitled?node-id=1-2&t=ahzumForPVQJBTq6-0)

- Левая панель: Monaco Editor с табами — [Template A ▼ движок] [Template B ▼ движок] [JSON]
- Каждый таб — независимый слот: своё тело шаблона + свой движок из селекта
- Правая панель верх (~2/3): Preview — синхронизируется с активным табом редактора
- Правая панель низ (~1/3): Метрики — avg / min / max / p95, размер вывода (bytes)

### Режим 1 — Editor mode (дефолт)

- Активный таб Template A → Preview показывает рендер Template A
- Активный таб Template B → Preview показывает рендер Template B
- Live preview с дебаунсом ~150ms при редактировании
- Метрики заполняются после Run Benchmark

### Режим 2 — Compare mode (после Run Benchmark или вручную)

- Preview разделён: левая половина — Template A, правая — Template B
- Метрики по обоим слотам side-by-side
- Вывод: `✓ Template B faster by 19%` / `✓ Output: identical` или `⚠ N differences`
- Сравнение любых комбинаций: HBS vs Pug, HBS vs HBS, Pug vs EJS — без ограничений

### Прочее

- Выбор N итераций для бенчмарка
- Подсветка синтаксиса в редакторе зависит от выбранного движка
- Добавить новый движок на бэке → фронт подхватывает автоматически

## 4. Управление шаблонами

- Публичная библиотека шаблонов?
- Пресеты и примеры для быстрого старта?
- Создание шаблона из sandbox или из личного кабинета
- Клонировать шаблон (свой или публичный → сохранить копию себе)

## 5. Персистентность состояния

- Состояние = `{ json }`
- **Для всех**: автосохранение в `localStorage` (прозрачно, без действий пользователя)
- **Для авторизованных**: сохранение на бэкенд → короткая ссылка `/s/{id}` (например)
- Приоритет восстановления при загрузке:
  1. URL `/s/{id}` → запросить с бэка
  2. URL `?state=base64` → восстановить из параметра
  3. `localStorage` → восстановить последнее
  4. Ничего → чистый редактор с пресетом

## 6. Share

- **Авторизованные**: сохранить состояние на бэкенд → `/s/{id}`, флаг `private` (только владелец) или публичная (любой)
- **Неавторизованные**: Share → `?state=base64` (состояние в URL) (или ту мач?), предупреждение: "Войдите, чтобы ссылка была постоянной"
- **Result-ссылка**: ссылка на конкретный RenderRun/бенчмарк — статичная страница с метриками
