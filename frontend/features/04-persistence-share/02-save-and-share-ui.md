# 04-02 Save, Share UI и SharedStatePage

## Цель

Кнопка Save/Share в action bar только для авторизованных пользователей. SharedStatePage для загрузки состояния по `/s/:id`.

## Зачем

Сохранение и шаринг — привилегия авторизованных. Гости работают в sandbox, но не видят кнопку Share — незачем предлагать функцию, которой нет. Состояние либо приходит с бэкенда (по ссылке `/s/:id`), либо пользователь работает с нуля.

## Use Cases

**Авторизованный пользователь:**
- Нажимает Save/Share — состояние сохраняется на бэкенд, URL меняется на `/s/{id}`, ссылка копируется в clipboard, snackbar "Saved! Link copied"
- Иконка кнопки Save отличается: несохранённые изменения (`isDirty`) — одна иконка, сохранено — другая

**Неавторизованный пользователь:**
- Кнопка Save/Share отсутствует в action bar — не скрыта через disabled, просто не рендерится

**SharedStatePage (`/s/:id`):**
- Пользователь (любой, авторизованный или нет) открывает ссылку — spinner, состояние загружается с бэкенда, редирект на `/sandbox`
- ID не существует или ошибка сети — страница "State not found" с кнопкой "Go to Sandbox"

**Fork при сохранении чужого состояния (неявный clone):**
- Открытие `/s/:id` сбрасывает `savedStateId = null` — sandbox "не знает" об оригинале
- Авторизованный пользователь нажимает Save → создаётся **новое** состояние (POST), URL меняется на `/s/{newId}`
- Оригинал (`/s/:id`) никогда не перезаписывается — Save всегда создаёт копию
- Никакого UI-подтверждения "вы форкаете чужое" не показывается — поведение прозрачное
- Исключение: если пользователь снова открыл **свою** ссылку и не редактировал — Save создаст ещё одну копию (это нормально, не баг)

## Критерии готовности

- [ ] Кнопка Save/Share рендерится только для `auth.isAuthenticated === true`
- [ ] Нажатие → POST на бэкенд → URL меняется на `/s/{id}` (без перезагрузки) → ссылка в clipboard
- [ ] `store.isDirty` отражается в иконке кнопки
- [ ] SharedStatePage: spinner при загрузке, redirect после успеха
- [ ] SharedStatePage: "State not found" при ошибке с кнопкой-fallback
- [ ] Нет base64 URL, нет `?state=` параметров нигде
