# 04-01 localStorage autosave и restore chain

## Цель

Автоматическое сохранение состояния sandbox в localStorage и цепочка восстановления при загрузке страницы.

## Зачем

Пользователь не должен терять работу при обновлении вкладки. Autosave прозрачен — не требует действий. Restore chain однозначна: сначала проверяем backend-сохранение (если пришли по ссылке), потом localStorage, потом дефолтный пресет.

## Use Cases

- Пользователь набрал шаблон, обновил вкладку — при открытии `/sandbox` код восстановлен из localStorage
- Пользователь открывает `/s/abc123` — состояние загружается с бэкенда, localStorage игнорируется
- Пользователь открывает `/sandbox` впервые — нет ни `/s/:id`, ни localStorage — загружается дефолтный пресет (Hello World)
- Restore chain порядок: `/s/:id` → localStorage → пресет

## Критерии готовности

- [ ] Autosave срабатывает через ~1s после последнего изменения в sandbox store
- [ ] При открытии `/sandbox` без параметров — восстанавливается из localStorage, если есть
- [ ] При открытии `/s/:id` — загружается с бэкенда, localStorage не перезаписывается до изменений пользователем
- [ ] При пустом localStorage — загружается дефолтный пресет
- [ ] Ошибка чтения localStorage (квота, невалидный JSON) — тихий fallback на пресет
- [ ] Reset/New sandbox очищает localStorage (`localStorage.removeItem('sandbox_state')`) и загружает пресет

## Поведение при множестве вкладок

**Решение для v1: last writer wins.**

- Все вкладки пишут в один ключ `sandbox_state`
- При обновлении любой вкладки восстанавливается последнее записанное состояние (из любой вкладки)
- Данные не теряются — просто восстанавливается то, что последним записала любая из вкладок
- Per-tab изоляция (`sandbox_state_<tabId>`) — future improvement, не в v1
